# One Overlay Window per display, rebuilt on display change

The crosshair is rendered with one borderless, transparent, click-through window per `NSScreen` — each drawing only the crosshair segment crossing its display — rather than a single window spanning the union of all display frames. We chose per-screen windows because they give correct per-display backing-scale rendering (Retina vs non-Retina), avoid the "dead gap" and mixed-scale-factor problems of a spanning window, and handle monitor hot-plug/rearrangement cleanly. On any `didChangeScreenParametersNotification` (debounced to coalesce bursts) we tear down and rebuild the entire window set rather than diffing it — display reconfiguration is rare and user-initiated, so the momentary rebuild flicker is acceptable and it eliminates a whole class of incremental-diffing bugs that make multi-monitor tools flaky.
